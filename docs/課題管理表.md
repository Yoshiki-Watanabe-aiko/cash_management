# 課題管理表：統合資産・経費管理システム (cash_management)

課題は通し番号で管理し、解決済みでも削除せず対応履歴を追記していく。旧 `docs/requirements.md` 9章「未確定・今後検討する拡張ポイント」から移行した（2026-07-10）。

## #1 マネーフォワードMEスクレイピング自動化（第二段階）
- 状態: 未解決
- 内容: 現状はMFMEの明細CSVエクスポート機能を手動配置する方式（第一段階）。Playwright等によるログイン自動化・エクスポート自動化を第二段階として検討する。具体的な実装方式・エラー時のリトライ/手動介入フローが未確定。
- 対応履歴:
  - 2026-07-XX（Phase 3）: 第一段階（CSV手動配置＋バッチ読込）を実装。詳細は[詳細設計書/01_データ収集パイプライン設計.md](詳細設計書/01_データ収集パイプライン設計.md)。

## #2 正規表現パース失敗時の低コストLLMフォールバック導入要否
- 状態: 未解決
- 内容: カードメールパーサーが正規表現で抽出に失敗した場合、Claude Haiku等の低コストLLMにフォールバックさせるかどうかが未確定。カード会社別パーサー本体が未実装（#3参照）のため、導入要否の判断もそれに従い保留中。

## #3 クレジットカード各社の速報メールサンプル未入手
- 状態: 未解決
- 内容: IMAP取込・重複防止・パーサーレジストリのフレームワークはPhase 4で実装済みだが、実際の利用速報メールサンプルが未入手のため、カード会社別`CardEmailParser`本体（正規表現による日付・金額・店舗名抽出）が未実装（レジストリは空）。サンプル入手後は対象金融機関の`m_institutions.card_alert_sender_email`に送信元アドレスを設定し、`app/services/card_parsers/`に実装を追加してレジストリへ登録する。
- 対応履歴:
  - 2026-07-XX（Phase 4）: フレームワーク先行実装（[ADR 0010](adr/0010-card-email-import-framework-first.md)）。パーサー本体は未着手。

## #4 マネーフォワードME資産評価CSVの列名マッピング未検証
- 状態: 未解決
- 内容: 資産評価CSVの実サンプルが未入手のため、`app/services/mf_assets_csv.py`の列名エイリアスは一般的に知られている表記から推測している。実サンプル入手後に列名マッピングの検証・調整が必要（証券の銘柄種別[国内株式/投資信託]判定もキーワードヒューリスティックであり要検証、[ADR 0009](adr/0009-moneyforward-account-name-column.md)関連）。

## #5 外貨建て資産の原通貨・為替レート分解管理
- 状態: 未解決
- 内容: 外貨建て資産（米国株・海外ETF等）は現時点で未保有だが将来保有の可能性が高い。当面は円換算後の評価額のみ保存し、原通貨・為替レートの分解管理はスコープ外とする。

## #6 現金払い等、自動取込元のない取引の手動新規登録機能
- 状態: 未解決
- 内容: 現金（財布）払いなど、メール・CSVいずれの自動取込元も持たない取引を手動で新規登録する機能は未実装。既存の手動補正画面はカテゴリ・事業按分率の編集のみで、取引そのものの新規作成には対応していない。

## #7 過去データの遡及取込
- 状態: 未解決
- 内容: 稼働開始前の過去データは遡って取り込まない方針（稼働開始日時点の残高を`opening_balance`として設定し、そこから先を追跡する）。将来的に遡及取込のニーズが出た場合の方式は未検討。

## #8 振替リンクの解除手段がフロントエンドに存在しない
- 状態: 解決済み（2026-07-09、Phase 10）
- 内容: `DELETE /api/transfers/{id}`はPhase 6で実装済みだったが、既存リンクの`transfer_id`を取得できる一覧系APIが存在しないため、フロントエンドから解除する手段がなかった（Phase 7時点では新規リンク作成のみ）。
- 対応履歴:
  - 2026-07-08（Phase 7）: 新規リンク作成UIのみ実装。解除手段は次フェーズへ持ち越しと明記。
  - 2026-07-09（Phase 10）: `GET /api/transfers`一覧APIと`LinkedTransfersPanel.tsx`解除UIを追加し解消。詳細は[詳細設計書/05_マスタデータ管理機能設計.md](詳細設計書/05_マスタデータ管理機能設計.md)。
