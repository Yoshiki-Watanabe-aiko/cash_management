# 振替の自動検知条件・集計除外・マッチング窓の営業日判定

振替相殺は「金額完全一致」「営業日ベースで0〜3日以内」「摘要への名義/口座名マッチ」の3条件をすべて満たすペアのみ自動リンクする。名義が摘要に現れないケースは自動検知の対象外とし、手動補正画面でのリンクに委ねる（無理な自動化による誤検知を避けるため）。日次バッチは「直近7日以内かつ未リンクの取引」を対象に毎回再評価し、既にリンク済みの取引は除外する。`t_transfers`にリンクされた取引は、キャッシュフロー・カテゴリ別支出・予算消化率など、あらゆる集計から除外する。

営業日判定は`jpholiday`等のライブラリで日本の祝日を算出し、年末年始の銀行休業期間（12/31〜1/3、毎年固定）はコード内にハードコードする。これにより運用者による年次のカレンダー更新作業を発生させずに、長期休暇期間の振替遅延にも対応できる。
