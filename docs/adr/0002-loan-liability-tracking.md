# ローン（負債）はMF自動連携を試し、失敗したら口座単位で恒久的に手動運用へ固定する

三井住友銀行・PayPay銀行のローン残高（200〜300万円規模）も資産管理の対象に含める。ローンはマネーフォワードMEでの自動連携を第一候補とするが、連携できるかは導入時点でしか分からない。日次バッチのたびに自動/手動を動的判定する案もあったが、ローン残高は月次更新程度の低頻度データであり、動的フォールバックの複雑さに見合わないと判断した。そのため `m_accounts` に取得方式を表すフラグを持たせ、導入時に一度だけ判定し、以降は口座ごとに固定する方式を採用した。連携状況が変わった場合は運用者が手動でフラグを切り替える。
