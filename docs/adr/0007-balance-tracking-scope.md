# 残高追跡の対象は銀行・証券・ローン・チャージ式QR決済のみ

純資産の集計対象となる「残高追跡対象口座」は、日次で資産スナップショットを持つ。銀行口座・チャージ式QR決済（PayPay/楽天Pay）は「初期残高＋取引累積」方式で、証券口座はマネーフォワードMEの資産評価CSVから、ローンはマネーフォワードME連携を第一候補とし失敗時は口座単位で手動運用に固定する（ADR 0002）。

クレジットカードと都度引き落とし式QR決済は対象外とした。理由は、クレカの取引データは利用速報メール（使った時点の通知）のみで、返済（引き落とし）は銀行口座側の出金としてしか記録されず、クレカ口座単体では残高が減ることなく増え続けてしまうため。クレカ・都度引き落とし式QR決済は支出のカテゴリ分類・事業按分のためだけに取引を記録し、`m_accounts`の残高追跡フラグは立てない。
