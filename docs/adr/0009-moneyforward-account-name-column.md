# MF CSVの口座表示名解決にm_accounts.moneyforward_account_nameを追加する

マネーフォワードME「収入・支出詳細」CSVおよび資産評価CSVの各行には、口座を示す表示名（「保有金融機関」列等）が含まれるが、この文字列はユーザーがMF側で設定した任意のニックネームであり、`m_accounts.account_name`（例:「楽天銀行 個人」）と一致するとは限らない。設定ファイルによる別建てのマッピング（YAML/JSON等）で解決する案もあったが、本プロジェクトはマスタデータをすべてDBの`m_`テーブルに集約する方針であるため、`m_accounts`に`moneyforward_account_name`（nullable）を追加し、CSVの表示名との突き合わせに使う設計とした。値が未設定の口座は`account_name`をそのまま照合対象とするフォールバックにより、既存口座への影響はない。

一致しない行（未登録の口座やタイプミス）はエラーにせず、`account_id`をNULLのまま`t_transactions`へ保存し（DDLで元々nullable）、未解決の口座表示名一覧をバッチサマリーに含める。これにより取込自体は継続しつつ、手動補正画面（7章）での事後対応に委ねる。
