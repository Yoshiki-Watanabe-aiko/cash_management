# 日次バッチの障害耐性方針

各金融機関の取込処理は機関ごとに独立してtry/exceptで囲み、1社の失敗が他社の処理をブロックしないようにする。軽量なリトライ（3回・数秒間隔）を機関ごとの処理に組み込み、一時的なネットワークエラーを無用なエラー通知に発展させない。機関ごとの結果は`t_batch_logs`にJSONB列（`institution_results`）として保存し、Discord通知でも機関別に成否を明記する。

DB接続自体に失敗する致命的なケースでは`t_batch_logs`へ記録できないため、バッチのエントリポイントを最上位のtry/exceptで囲み、DBを経由しない簡易Discord通知（「バッチが起動に失敗しました」）を別経路で送る。

Windowsタスクスケジューラには「スケジュールされた時刻を逃した場合はできるだけ早く実行する」「タスクを実行するためにスリープを解除する」を設定し、PCがスリープ中でも自動実行されるようにする。完全シャットダウンされていた場合に備え、ログオン時/起動時トリガーも保険として追加する。
